04 — Riscos e Limitações

O que pode travar o crescimento

🔴 B6 — Autorização incompleta nas rotas da API
Rotas como /api/tcc/[id]/messages podem não verificar se o TCC pertence ao usuário autenticado. Um usuário mal-intencionado poderia acessar dados de outro. Prioridade máxima antes de abrir o cadastro público.
🔴 Nenhuma política de privacidade ou termos de uso
A LGPD (Lei 13.709/2018) exige, no mínimo, política de privacidade publicada, base legal para tratamento de dados e mecanismo de exclusão. Operar sem isso é ilegal no Brasil.
🟠 Sem rate limiting nas APIs de IA
Qualquer usuário (ou script) pode chamar /api/chat e /api/gerar-tcc sem limitação no servidor. Custo de API pode explodir com poucos usuários abusivos.
🟠 B5 — Limite diário em localStorage
O contador de mensagens diárias é gravado no navegador. Qualquer usuário técnico pode burlar abrindo o DevTools e limpando o localStorage. Mover para o banco de dados antes de escalar.
🟠 Índice de plágio exibido sem ser real
O painel exibe um "% de similaridade" que é calculado localmente sem nenhuma verificação real. Se um estudante ou professor confiar nesse número, isso é um risco de credibilidade e possivelmente de responsabilidade legal.
🟠 Nenhum serviço de e-mail integrado
Sem verificação de e-mail, qualquer pessoa pode criar conta com um e-mail inválido. Sem recuperação de senha, usuários que perdem o acesso ficam bloqueados. Bloqueia retenção de usuários.
🟢 Next.js apenas — gargalo para IA avançada
API Routes do Next.js rodam em serverless functions com timeout de ~10s (Vercel). Pipelines de RAG, geração longa e embeddings precisam de tempo de execução maior. Aceitável agora, problema em breve.
🟢 Zero testes automatizados
Não há nenhum arquivo de teste no repositório. Mudanças de código podem quebrar funcionalidades sem que o time perceba antes dos usuários.

Recomendação: Manter Next.js puro até o fim do beta e primeiros pagantes. A partir do momento em que RAG com PDFs, geração de capítulos longos ou processamento assíncrono forem necessários, migrar para Next.js + FastAPI. O FastAPI tem tipagem nativa, documentação automática (Swagger), asyncio nativo e integra perfeitamente com LangChain, LlamaIndex e toda a stack Python de IA.
06 — Mercado Brasileiro

Oportunidades específicas para o Brasil

Conformidade acadêmica
Verificação ABNT automática
Checagem real de formatação (margens, fonte, espaçamento, citações ABNT NBR 6023) ao exportar. Diferencial competitivo forte frente a ferramentas genéricas como ChatGPT.
Estrutura por tipo de trabalho
Templates por gênero acadêmico
Estruturas específicas para TCC, monografia, artigo científico, relatório de estágio, dissertação e tese — cada um com suas seções obrigatórias.
Orientação e parceria
Painel do orientador
Permitir que o professor orientador acesse o trabalho, faça comentários e acompanhe o progresso. Abre uma linha de receita B2B (instituições, coordenações).
Referências acadêmicas
Ingestão de PDFs e RAG
Permitir que o estudante suba artigos e livros. A IA cita diretamente esses documentos. Reduz alucinação e aumenta confiança no produto. (não confirmado — requer FastAPI)
Originalidade e integridade
Integração com verificadores
Integração com Turnitin, Copyleaks ou Compilatio — serviços usados pelas universidades brasileiras. Substituiria o índice simulado atual. (não confirmado)
Cursos e normas
Fluxos por área do conhecimento
Direito pede estrutura diferente de Engenharia ou Psicologia. Prompts e templates específicos por área são um diferencial relevante para conversão.
Exportação acadêmica
Word e LaTeX com formatação ABNT
Exportar para .docx com estilos ABNT aplicados automaticamente seria um diferencial forte. Hoje só PDF via html2canvas (qualidade limitada).
Proteção de dados
Conformidade com LGPD
Política de privacidade publicada, consentimento explícito, direito de exclusão e exportação dos dados pessoais. Obrigatório por lei para operar no Brasil.

1 a 3 meses

  • Histórico de versões do documento
  • Templates de estrutura por tipo de trabalho
  • Exportação Word (.docx) com estilos ABNT
  • Separar backend em FastAPI (piloto de IA)

  • Painel do orientador (MVP)
  • Verificação ABNT automática na exportação
  • Testes automatizados (mínimo: fluxo crítico)

Longo prazo
3 a 12 meses

  • RAG avançado com banco vetorial
  • Integração com verificador de originalidade real
  • Plano institucional (B2B — universidades)
  • Detecção de lacunas de citação
  • Colaboração em tempo real (co-autoria)
  • Timeline de evolução do TCC
  • App mobile (React Native / PWA)
  • API pública para integrações de terceiros

Visão por maturidade do produto

MVP real
Estado atual + segurança + LGPD
O core loop já existe. O MVP real é corrigir os riscos que impedem o cadastro público: B6, rate limiting e política de privacidade.
Versão vendável
E-mail + Word + ABNT + orientador
Com e-mail funcional, exportação Word de qualidade e painel do orientador, o produto tem proposta de valor clara para pagar e recomendar.
Versão defensável
RAG + verificador real + B2B
RAG com PDFs do usuário e verificação de originalidade real criam barreiras competitivas que ferramentas genéricas não conseguem replicar facilmente.
08 — Contribuição dos Sócios

Como contribuir sem programar

Pesquisa com usuários

  • Recrutamento de 5 a 10 estudantes em fase de TCC
  • Entrevistas semi-estruturadas (30 min)
  • Registro de dores, expectativas e linguagem usada
  • Documentar no Notion: insights por perfil de aluno

Benchmark de concorrentes

  • Mapear ferramentas similares no Brasil e no exterior
  • Testar funcionalidades e registrar diferenciais
  • Comparar preços, planos e proposta de valor
  • Identificar o que o Teseo faz que nenhum outro faz

QA — Testes de produto

  • Testar todos os fluxos com um estudante real como guia
  • Registrar bugs, confusões e pontos de abandono
  • Validar fluxo de pagamento (modo teste)
  • Documentar problemas em Trello com print + descrição

Conformidade e documentos legais

  • Rascunho da política de privacidade (LGPD)
  • Rascunho dos termos de uso
  • Consulta com advogado especializado (recomendado)
  • Definição de base legal para tratamento dos dados

Relacionamento acadêmico

  • Contato com professores orientadores para feedback
  • Parceria com coordenações de curso para piloto
  • Identificar influenciadores em comunidades de estudantes
  • Proposta de parceria com bibliotecas universitárias

Conteúdo e comunicação

  • Revisão e melhoria dos textos da landing page
  • Criação de conteúdo para redes sociais (TikTok, Instagram)
  • FAQ expandido com dúvidas reais de estudantes
  • Guias de como usar o Teseo para cada tipo de trabalho
